# Trading Bot Execution Bridge API Documentation

The Execution Bridge connects the Python-based AI trading bot with external consumers (like dashboards or other applications) and MetaTrader 5 Expert Advisors (EAs).

**Base URL:** `http://<server-ip>:8000` (Default: `http://localhost:8000`)
**Interactive Swagger UI:** `http://localhost:8000/docs`

---

## Public Endpoints (No Authentication Required)
These endpoints are designed for external dashboards, monitoring tools, or web/mobile clients.

### 1. Get Current Signals
Retrieves all currently active trading signals generated by the bot.

- **URL:** `/signals/current`
- **Method:** `GET`
- **Response Format:** JSON
- **Success Response:** `200 OK`

**Example Response:**
```json
{
  "count": 2,
  "signals": [
    {
      "symbol": "EURUSD",
      "direction": "BUY",
      "confidence": 85.5,
      "entry_price": 1.0854,
      "stop_loss": 1.0820,
      "take_profit": 1.0910,
      "reasoning": "Strong bullish engulfing pattern at key support level with MACD crossover.",
      "age_seconds": 124,
      "is_stale": false
    },
    {
      "symbol": "GBPUSD",
      "direction": "SELL",
      "confidence": 78.0,
      "entry_price": 1.2650,
      "stop_loss": 1.2680,
      "take_profit": 1.2580,
      "reasoning": "Bearish divergence on RSI near major resistance.",
      "age_seconds": 4530,
      "is_stale": false
    }
  ],
  "last_updated": "2026-03-19T22:15:30.123456+00:00"
}
```
*Note:* The `is_stale` flag becomes `true` if a signal is older than 2 hours (7200 seconds). Signals are sorted by confidence (highest first).

### 3. Get Signal History
Retrieves recent historical signals generated by the bot.

- **URL:** `/signals/history`
- **Method:** `GET`
- **Query Parameters:**
  - `limit` (optional): Integer, default `50`. Maximum number of recent signals to return.
- **Success Response:** `200 OK`

**Example Response:**
```json
{
  "count": 50,
  "total": 142,
  "signals": [
    {
      "symbol": "EURUSD",
      "direction": "BUY",
      "confidence": 85.5,
      "entry_price": 1.0854,
      "stop_loss": 1.0820,
      "take_profit": 1.0910,
      "reasoning": "Strong bullish engulfing...",
      "timestamp": 1710886530.0,
      "received_at": 1710886531.5
    }
    // ... more historical signals
  ]
}
```

### 4. Health Check
Simple endpoint to verify the API server is up and responsive.

- **URL:** `/health`
- **Method:** `GET`
- **Success Response:** `200 OK`

**Example Response:**
```json
{
  "status": "ok",
  "uptime_seconds": 86400,
  "signals_tracked": 3
}
```

## MT5 Bridge Endpoints (Used by MetaTrader 5 EA)
These endpoints are typically polled by the MT5 Expert Advisor to execute trades. No API key is required so the EA can easily consume them.

### 5. Poll for New Pending Signal
Polls for a pending trade execution signal for a specific symbol. *Note: Fetching a pending signal removes it from the pending execution queue (it "pops" it).*

- **URL:** `/poll/{symbol}`
- **Method:** `GET`
- **Path Parameters:**
  - `symbol` (e.g., `EURUSD`)
- **Success Response:** `200 OK`

**Example Response (New Trade):**
```json
{
  "status": "new_trade",
  "direction": "BUY",
  "entry": 1.0854,
  "sl": 1.0820,
  "tp": 1.0910
}
```

**Example Response (No Pending Trade):**
```json
{
  "status": "no_signal"
}
```

**Example Response (Trade Expired):**
```json
{
  "status": "expired"
}
```
*(Signals older than 5 minutes expire and won't be executed).*

### 6. Get Latest Analysis for Symbol
Retrieves the most recent analysis for a specific symbol (similar to `/signals/current` but for a single symbol, without popping it from any queue).

- **URL:** `/analysis/{symbol}`
- **Method:** `GET`
- **Path Parameters:**
  - `symbol` (e.g., `EURUSD`)
- **Success Response:** `200 OK`

**Example Response:**
```json
{
  "status": "success",
  "symbol": "EURUSD",
  "direction": "BUY",
  "confidence": 85.5,
  "reasoning": "Strong bullish engulfing pattern...",
  "entry": 1.0854,
  "sl": 1.0820,
  "tp": 1.0910,
  "age_seconds": 124
}
```

## Internal Endpoints (API Key Required)
These endpoints are exclusively used by the Python trading bot (`main.py`) to push data into the server. 

**Authentication:** 
Must include header: `X-API-KEY: <your-secret-key>` (Configured via `EXECUTION_BRIDGE_KEY` in `.env`).

### 7. Post Signal
Pushes a new trade signal from the AI engine to the server.

- **URL:** `/signals`
- **Method:** `POST`
- **Request Body:** JSON payload of `TradeSignal`
```json
{
  "symbol": "EURUSD",
  "direction": "BUY",
  "entry_price": 1.0854,
  "stop_loss": 1.0820,
  "take_profit": 1.0910,
  "confidence": 85.5,
  "reasoning": "Strong bullish momentum...",
  "timestamp": 1710886530.0
}
```
- **Success Response:** `200 OK`
```json
{
  "status": "queued",
  "symbol": "EURUSD"
}
```

### 8. Post Bot Heartbeat
Pushes bot operational status at the end of every scan cycle.

- **URL:** `/bot/heartbeat`
- **Method:** `POST`
- **Request Body:** JSON payload of `BotHeartbeat`
```json
{
  "last_scan_time": "2026-03-19T22:18:45.000000+00:00",
  "symbols_scanned": ["EURUSD", "GBPUSD"],
  "scan_interval": 60,
  "timeframes_analyzed": ["1h", "4h"],
  "mode": "live",
  "total_signals_generated": 142
}
```
- **Success Response:** `200 OK`
```json
{
  "status": "ok"
}
```
